This repository contains the embedded control firmware for the autonomous buggy project. The architecture is engineered around a dual $ O(1) $ deterministic Finite State Machine (FSM) implemented on a Microchip ds\+PIC33 microcontroller. It provides predictable, real-\/time behavior without using conditional switch-\/cases or blocking delays.

To ensure strict real-\/time execution safety, the system separates {\bfseries{State Transitions}} (triggered by spontaneous external events) from {\bfseries{State Actions}} (continuous operations running at fixed frequencies) using two flat functional pointer tables with $ O(1) $ computational complexity.\hypertarget{md__r_e_a_d_m_e_autotoc_md3}{}\doxysubsection{\texorpdfstring{1. Dual $ O(1) $ Optimization Mechanics}{1. Dual $ O(1) $ Optimization Mechanics}}\label{md__r_e_a_d_m_e_autotoc_md3}

\begin{DoxyItemize}
\item {\bfseries{The Transition Matrix ({\ttfamily transition\+\_\+table}):\+}} A 2D array of function pointers indexed directly by {\ttfamily \+[Current\+\_\+\+State]\+\+[Incoming\+\_\+\+Event]\+}. It updates the state machine variables instantaneously upon event arrival.
\item {\bfseries{The Cyclic Action Array ({\ttfamily cyclic\+\_\+action\+\_\+table}):\+}} A flat array of function pointers indexed directly by {\ttfamily \+[Current\+\_\+\+State]\+}. The main scheduler executes the behavior of the current active state via a single calculated jump ({\ttfamily call}) instruction, eliminating variable-\/time {\ttfamily switch-\/case} evaluation paths.
\end{DoxyItemize}

\hypertarget{md__r_e_a_d_m_e_autotoc_md5}{}\doxysection{\texorpdfstring{Finite State Machine Specification}{Finite State Machine Specification}}\label{md__r_e_a_d_m_e_autotoc_md5}
The system processes three distinct functional operational states:\+\hypertarget{md__r_e_a_d_m_e_autotoc_md6}{}\doxysubsection{\texorpdfstring{1. STATE\+\_\+\+HALTED}{1. STATE\+\_\+\+HALTED}}\label{md__r_e_a_d_m_e_autotoc_md6}

\begin{DoxyItemize}
\item {\bfseries{Entry /\+ Behavior:\+}} The robot starts initialization in this state.
\item {\bfseries{Motor Actuation:\+}} The PWM Duty Cycle (DC) of all four H-\/\+Bridge control signals (RD1–\+RD4) is set strictly to 0\%.
\item {\bfseries{Indicator Configuration:\+}} The left-\/side (RB8) and right-\/side (RF1) lights flash synchronously at a frequency of 1 Hz. Low-\/intensity auxiliary lights (RG1) are forced OFF.
\item {\bfseries{Transitions:\+}} Pressing the physical button RE8 switches the system to {\ttfamily STATE\+\_\+\+MOVING}.
\end{DoxyItemize}\hypertarget{md__r_e_a_d_m_e_autotoc_md7}{}\doxysubsection{\texorpdfstring{2. STATE\+\_\+\+MOVING}{2. STATE\+\_\+\+MOVING}}\label{md__r_e_a_d_m_e_autotoc_md7}

\begin{DoxyItemize}
\item {\bfseries{Motor Actuation:\+}} The 10 k\+Hz motor PWM duty cycles on pins RD1–\+RD4 are adjusted dynamically inside a 500 Hz control loop matching the current speed and yaw rate references received from the UART interface:\+   \[  left\_pwm = speed - yawrate
\]   \[  right\_pwm = speed + yawrate
\] Saturation limits are locked strictly between $ -100\% $ and $ +100\% $.
\item {\bfseries{Indicator Configuration:\+}} Left and right-\/side lights are turned completely OFF. Low-\/intensity auxiliary lights are turned ON.
\item {\bfseries{Transitions:\+}} \texorpdfstring{$\ast$}{*} Pressing button RE8 drops the system back into {\ttfamily STATE\+\_\+\+HALTED}.
\begin{DoxyItemize}
\item If the 500 Hz IR sensor subsystem (AN14/\+\+AN15) senses an obstacle within the critical threshold range, the firmware immediately moves into {\ttfamily STATE\+\_\+\+OBSTACLE\+\_\+\+AVOIDANCE}.
\end{DoxyItemize}
\end{DoxyItemize}\hypertarget{md__r_e_a_d_m_e_autotoc_md8}{}\doxysubsection{\texorpdfstring{3. STATE\+\_\+\+OBSTACLE\+\_\+\+AVOIDANCE}{3. STATE\+\_\+\+OBSTACLE\+\_\+\+AVOIDANCE}}\label{md__r_e_a_d_m_e_autotoc_md8}

\begin{DoxyItemize}
\item {\bfseries{Behavior Maneuver:\+}} The robot halts forward instruction and executes an automated, non-\/blocking time-\/tracked hardware maneuver:\+
\begin{DoxyEnumerate}
\item Rotate 90 degrees clockwise.
\item Advance forward continuously for exactly 2.\+0 seconds.
\item Rotate 90 degrees counter-\/clockwise to realign with the initial bearing angle.
\end{DoxyEnumerate}
\item {\bfseries{Indicator Configuration:\+}} The right-\/side lights blink at 1 Hz, while left-\/side lights stay dark and low-\/intensity lights remain active.
\item {\bfseries{Evaluation Loop Retry Logic:\+}} After the 2-\/second directional maneuver completes, the system samples the IR sensor again.
\begin{DoxyItemize}
\item If the path is now clear, it returns to {\ttfamily STATE\+\_\+\+MOVING}.
\item If the obstacle remains, the recovery sequence repeats for a maximum limit of {\bfseries{3 attempts}}. If the path remains obstructed after the third attempt, the system enters the safe {\ttfamily STATE\+\_\+\+HALTED} fallback state.
\end{DoxyItemize}
\item {\bfseries{Transitions:\+}} Pressing button RE8 at any point instantly stops the sequence and shifts the machine to {\ttfamily STATE\+\_\+\+HALTED}.
\end{DoxyItemize}

\hypertarget{md__r_e_a_d_m_e_autotoc_md14}{}\doxysection{\texorpdfstring{Firmware Integrity \& Safe Integration}{Firmware Integrity \& Safe Integration}}\label{md__r_e_a_d_m_e_autotoc_md14}

\begin{DoxyEnumerate}
\item {\bfseries{Non-\/\+Blocking Execution Rules:\+}} Software timing loops must never utilize blocking delay functions (e.\+g., {\ttfamily \+\_\+\+\_\+delay\+\_\+ms()}). Timing tracks for structural state tasks (such as the 2-\/second avoidance maneuver) must evaluate increments computed within the periodic 500 Hz Timer Interrupt.
\item {\bfseries{Interrupt Data Guardrails:\+}} Any telemetry arrays or state structures shared between the main execution thread and the UART/\+\+Timer Peripheral Interrupt Service Routines (ISRs) must be declared with the {\ttfamily volatile} qualifier.
\item {\bfseries{Atomic Operations:\+}} To ensure total data integrity, wrap data exchanges or state mutations inside brief critical section blocks using assembly built-\/ins ({\ttfamily \+\_\+\+\_\+builtin\+\_\+disi(0x3\+FFF)}) to temporarily disable nested interrupts during shared buffer modifications. 
\end{DoxyEnumerate}

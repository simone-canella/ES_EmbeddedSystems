\hypertarget{index_autotoc_md0}{}\doxysection{\texorpdfstring{ds\+PIC33 Buggy Control Firmware}{ds\+PIC33 Buggy Control Firmware}}\label{index_autotoc_md0}
This repository contains the embedded control firmware for an autonomous buggy project. The architecture relies on a dual \$\+O(1)\$ deterministic Finite State Machine (FSM) running on a Microchip ds\+PIC33 microcontroller to provide predictable, real-\/time behavior without utilizing conditional switch-\/cases or blocking delays.

\hypertarget{index_autotoc_md2}{}\doxysubsection{\texorpdfstring{System Architecture}{System Architecture}}\label{index_autotoc_md2}
To guarantee real-\/time safety, the firmware completely isolates spontaneous state transitions from continuous fixed-\/frequency state actions.


\begin{DoxyItemize}
\item {\bfseries{The Transition Matrix ({\ttfamily \doxylink{fsm_8c_a27d5b9351e8b1b15cc1fa3dd9f6998c0}{state\+\_\+table}})}}:\+ A 2D function pointer array indexed directly by {\ttfamily \+[Current\+\_\+\+State]\+\+[Incoming\+\_\+\+Event]\+} updates the system state instantaneously in \$\+O(1)\$ computing time upon event arrival.
\item {\bfseries{Cooperative Scheduler}}:\+ A time-\/sliced task engine driven by a 500 Hz TIMER1 heartbeat executes registered subsystem callback functions non-\/preemptively when their configured tick periods elapse.
\end{DoxyItemize}

\item {\bfseries{Behavior}}:\+ The buggy initializes in this safe resting state with all four H-\/\+Bridge PWM duty cycles set strictly to 0\%.

\item {\bfseries{Transitions}}:\+ Pressing the physical RE8 button shifts the machine context into {\ttfamily \doxylink{fsm_8h_ae2946618e700f814e6147195c808f4c8ae8fbb3519d77b1f430b38756458dcad6}{STATE\+\_\+\+MOVING}}.

\item {\bfseries{Motor Actuation}}:\+ Wheels are driven dynamically at a 10 k\+Hz PWM frequency. Duty cycles are calculated and updated inside a 500 Hz control loop based on UART velocity and yaw references.

\item {\bfseries{Transitions}}:\+

\begin{DoxyItemize}
\item {\bfseries{Behavior Maneuver}}:\+ The buggy abandons forward commands to execute a sequential, non-\/blocking hardware recovery maneuver:\+
\begin{DoxyEnumerate}
\item {\bfseries{Stop}}:\+ The vehicle zeroes actuation and waits until accelerometer integration confirms total zero kinetic inertia.
\item {\bfseries{Rotate CW}}:\+ Applies yaw actuation until the gyroscope tracks a relative heading beyond -\/80.\+0 degrees (empirically estabilished).
\item {\bfseries{Move Forward}}:\+ Advances forward continuously for exactly 2.\+0 seconds (1000 loop ticks) or stops instantly if another obstruction is detected.
\item {\bfseries{Rotate CCW}}:\+ Applies counter-\/clockwise yaw actuation to realign the heading beyond +80.\+0 degrees.
\end{DoxyEnumerate}
\item {\bfseries{Indicators}}:\+ The right-\/side lights blink at 1 Hz to signal maneuvering, the left lights stay dark, and low-\/intensity lights remain active.
\item {\bfseries{Evaluation \& Retries}}:\+ After concluding the directional sequence, the IR sensor is sampled again. If the path is verified clear (\texorpdfstring{$>$}{>} 20.\+0 cm), the machine transitions back to {\ttfamily \doxylink{fsm_8h_ae2946618e700f814e6147195c808f4c8ae8fbb3519d77b1f430b38756458dcad6}{STATE\+\_\+\+MOVING}}.

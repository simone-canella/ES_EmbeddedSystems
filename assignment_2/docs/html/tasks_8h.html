Detailed Description

Subsystem task abstractions and hardware setup declarations for the robot buggy.

Author
Simone Canella, Samuele Isaia, Riccardo Parotelli
Version
1.0.0

This header file declares the initialization interfaces responsible for configuring low-level microcontroller peripherals and registering periodic execution tasks with the cooperative system scheduler.

void hw_init ( RobotState_t * rover)

Configures low-level microcontroller hardware registers and peripherals.

Initializes all necessary hardware peripherals and sets up initial robot state.

Parameters
roverPointer to the robot state structure to configure.
Returns

Sets up all required physical hardware components on the dsPIC core. This handles the initialization of data direction registers (TRIS), latch configurations (LAT), Peripheral Pin Select (PPS) pin remapping for UART/SPI communication links, analog-to-digital converters (ADC) for infrared and battery sensing, and Output Compare modules for the 10 kHz motor PWM waves.

void tasks_init ( void * param)

Registers all system runtime operational tasks into the scheduler matrix.

Initializes and registers all scheduler tasks.

Parameters
paramGeneric initialization parameter container representing the robot state.
Returns
void

Connects individual software tasks (such as sensor reading, state machine dispatching, motor actuation, and telemetry logging) with the periodic cyclic scheduler. In order to guarantee maximum execution time budgets for every single clock tick, tasks are registered with pre-calculated initial offsets so that their worst-case workloads do not overlap.

Registers all system runtime operational tasks into the scheduler matrix.

Parameters
paramPointer to the robot state structure to be passed into tasks.

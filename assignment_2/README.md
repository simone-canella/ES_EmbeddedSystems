# dsPIC33 Buggy Control Firmware
This repository contains the embedded control firmware for an autonomous buggy project. The architecture relies on a dual $O(1)$ deterministic Finite State Machine (FSM) running on a Microchip dsPIC33 microcontroller to provide predictable, real-time behavior without utilizing conditional switch-cases or blocking delays.

---

## System Architecture

To guarantee real-time safety, the firmware completely isolates spontaneous state transitions from continuous fixed-frequency state actions.

* **The Transition Matrix (`state_table`)**: A 2D function pointer array indexed directly by `[Current_State][Incoming_Event]` updates the system state instantaneously in $O(1)$ computing time upon event arrival.
* **Cooperative Scheduler**: A time-sliced task engine driven by a 500 Hz TIMER1 heartbeat executes registered subsystem callback functions non-preemptively when their configured tick periods elapse.

---

## Finite State Machine (FSM)

The system processes three primary functional operational states:

### 1. STATE_HALTED

* **Behavior**: The buggy initializes in this safe resting state with all four H-Bridge PWM duty cycles set strictly to 0%.
* **Indicators**: The left-side (RB8) and right-side (RF1) indicator lights flash synchronously at 1 Hz, while the low-intensity auxiliary lights (RG1) remain disabled.
* **Transitions**: Pressing the physical RE8 button shifts the machine context into `STATE_MOVING`.

### 2. STATE_MOVING

* **Motor Actuation**: Wheels are driven dynamically at a 10 kHz PWM frequency. Duty cycles are calculated and updated inside a 500 Hz control loop based on UART velocity and yaw references.
* **Indicators**: Both side indicator lights are turned OFF, while the low-intensity auxiliary lights are turned ON.
* **Transitions**:
* Pressing the RE8 button forces the system back to `STATE_HALTED`.
* By default inital linear and angular velocities are both set to zero. 
* To change the default speed the command $PCREF,XX,YY* where XX and YY are respectively linear and angular speed
* If the IR sensor polling loop detects an obstacle closer than the 35.0 cm safety threshold, the system dispatches an `EVENT_OBSTACLE_DETECTED` trigger, instantly pivoting to `STATE_OBSTACLE_AVOIDANCE`.

### 3. STATE_OBSTACLE_AVOIDANCE

* **Behavior Maneuver**: The buggy abandons forward commands to execute a sequential, non-blocking hardware recovery maneuver:
  1. **Stop**: The vehicle zeroes actuation and waits until accelerometer integration confirms total zero kinetic inertia.
  2. **Rotate CW**: Applies yaw actuation until the gyroscope tracks a relative heading beyond -80.0 degrees (empirically estabilished).
  3. **Move Forward**: Advances forward continuously for exactly 2.0 seconds (1000 loop ticks) or stops instantly if another obstruction is detected.
  4. **Rotate CCW**: Applies counter-clockwise yaw actuation to realign the heading beyond +80.0 degrees.


* **Indicators**: The right-side lights blink at 1 Hz to signal maneuvering, the left lights stay dark, and low-intensity lights remain active.
* **Evaluation & Retries**: After concluding the directional sequence, the IR sensor is sampled again. If the path is verified clear (> 20.0 cm), the machine transitions back to `STATE_MOVING`. If blocked, the maneuver increments a tracker and repeats up to a maximum limit of 3 attempts before triggering an emergency shutdown to `STATE_HALTED`.

## Real-Time Task Scheduler Matrix

Data collection and physical actions run concurrently at strict frequencies generated by the core 500 Hz hardware timer tick.

| Subsystem Task | Loop Frequency | Description |
| --- | --- | --- |
| **IR Sensor Polling** | 500 Hz | Refreshes the ADC buffer for distance metrics, subtracts specific sensor hardware offsets, and monitors the immediate path for threshold breaches. |
| **Movement Actuation** | 500 Hz | Evaluates active state requirements, processes avoidance sequences, and applies target output voltages to the H-Bridge. |
| **Button Polling** | 100 Hz | Debounces and evaluates the RE8 input pin for state toggles, and checks RE9 to trigger a diagnostic UART buffer transmission (`$MBUF`). |
| **IMU Spatial Fusion** | 100 Hz | Interfaces via SPI/I2C to read the Accelerometer, Magnetometer, and Gyroscope registers, running trigonometric integrations to map roll, pitch, and relative yaw. |
| **Bluetooth Telemetry** | 10 Hz | Formats and pushes NMEA spatial packages (`$MDIST`, `$MANGLE`) and high-level FSM diagnostics via the serial stream. |
| **Command Parsing** | Asynchronous | Unpacks and decodes asynchronous `$PCREF,speed,yawrate*` frames from the serial queue while enforcing safety saturation limits. |
| **Heartbeat & Logging** | 1 Hz | Alternates pin latches to display avoidance light codes, toggles the main-loop health LED (A0), and writes `$MBAT` battery voltages over UART. |

## H-Bridge Actuation Logic Mapping

Directional momentum is provided by four independent Output Compare PWM generators running continuously at a fixed 10 kHz base frequency.

| Direction Command | PWM Applied (Duty > 0) | Low-Level Ground Pin |
| --- | --- | --- |
| Left Wheels Forward | RD2 / RP66 | RD1 / RP65 = 0 |
| Left Wheels Backward | RD1 / RP65 | RD2 / RP66 = 0 |
| Right Wheels Forward | RD4 / RP68 | RD3 / RP67 = 0 |
| Right Wheels Backward | RD3 / RP67 | RD4 / RP68 = 0 |

## The bluetooth interface (Linux/macOS Native)
This interface routes the system's UART telemetry data wirelessly over a **Bluetooth connection instead of a physical cable link**. 

Because this interface relies on a shell script (`.sh`), it is natively compatible **only with POSIX operating systems like Linux and macOS**. Standard Windows environments (Command Prompt and PowerShell) cannot execute this file directly.

### Native Execution (Linux/macOS)
In the root folder of the project, grant execution permissions and run:

```bash
chmod +x ./bluetooth_interface/bt_interface.sh
./bluetooth_interface/bt_interface.sh
```
